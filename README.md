# ホテル清掃シフト最適化ダッシュボード

## プロジェクト概要
ホテル業界7年の実務経験をもとに、チェックアウト時間・客層・シーズンなどの
予約情報のダミーデータを作成しデータ分析。
清掃スタッフの最適なシフト配置を提案するダッシュボード。

## デモアプリ
https://hotelportfolio-fintjqdfzatqmgfhaetjsc.streamlit.app/

## データに関する注記
本プロジェクトは勤務先ホテルでの実務経験をモデルとし、
ダミーデータを作成・分析したものである。

料金設定・顧客情報などの機密情報は使用せず、
地方の小規模ホテルの一般的な料金帯を参考に設定している。

客層傾向・チェックアウト時間・稼働状況については
周辺ホテルとも共通する一般的な実務知見をもとに
ダミーデータとして再現している。

## 分析対象ホテルの想定
- 大分県の小規模旅館（全17室）
- 1階：和室（大）1室、2階：シングル・和室・ダブル・和洋室
- 近隣に百名山が3つあり登山客の拠点としても利用される
- ビジネス・観光・遠征・登山など多様な客層が宿泊

## 使用技術
- Python
- pandas
- matplotlib

- Streamlit

## 分析の背景
小規模ホテルでの実務経験から、清掃スタッフの人員配置は
現場感覚に頼ることが多く、人員過不足による
業務の滞り（清掃の遅れ・手待ち時間の発生）という
課題があった。
本プロジェクトでは、予約データを可視化することで
効率的なシフト設計の根拠を提供することを目的とする。
実データを用いることでより精度の高い提案が可能になる。

## 機能説明
- 月別・シーズン別・曜日別の予約件数の可視化
- チェックアウト時間帯・客層別の傾向分析
- 清掃スタッフの月別必要人数の算出
- 分析結果のまとめ表示

## 前提条件
- 清掃開始時間：8時以降
- スタッフ1人あたりの稼働時間：240分（休憩なし）
- 清掃所要時間は実務経験をもとに設定

## 分析結果・考察

### チェックアウト時間
8・10時にチェックアウトが集中しており、
清掃開始のピーク時間帯が分散している。
ピーク時の8時と10時に合わせた人員配置が重要。

### 曜日別傾向
土曜日に最も予約が集中。
平日はビジネス客、週末は観光・遠征・登山客が多い傾向。

### シーズン別傾向
通常期が件数は最も多いが、最繁忙期・繁忙期は
ほぼ満室状態のため、限られた時間での清掃効率が特に重要。

### 月別予約傾向
4〜5月（GW・登山シーズン）に予約が集中し、
2〜3月は閑散期として落ち込む傾向がある。

### 清掃スタッフ必要人数
4・5月（GW・登山シーズン）と10・11月（紅葉シーズン）に
必要人数が増加。繁忙期に合わせた事前の人員確保が重要。

### 客層別の清掃タイミング
- ビジネス客（7〜8時退室）→ 早朝清掃に対応
- 観光客（9〜10時退室）→ 10時出勤スタッフで対応
- 遠征客（8〜10時退室）→ 目的により退室時間にばらつきがあるため予約状況を
確認してシフトを調整
- 登山客（5〜7時退室）→ 清掃は8時以降

## 分析まとめ
- チェックアウトは8時・10時に集中 → 時差出勤が効率的
- 繁忙期（4-5月・10-11月）は2-3名体制を推奨
　→ GW・登山・紅葉シーズンで予約が集中し、登山客は早めだが観光客は遅めのチェックアウトのため
- 通常期は客層により1-2名で調整
　→ ビジネス客中心の日はチェックアウトが早いため
- 週次で予約状況を確認し柔軟にシフト調整することを推奨
　→ 予約状況は1週間単位で比較的安定しており、シフト設計の精度を保ちやすいため

## シフト最適化提案

本ダッシュボードは年間傾向の把握に活用。

### 必要人数の目安
- 通常日：1〜2名
- 閑散期（2〜3月・6月・9月）：1〜2名体制を推奨
※午前と午後に１名ずつ配置し、午後にはイレギュラー業務を行う
- 繁忙期（4〜5月・10〜11月）：2〜3名体制を推奨

### 時差出勤の提案
チェックアウトが8時と10時に集中しているため
全員同時出勤より時差出勤が効率的。

【推奨シフト例】
■ 通常日
- 基本：8時出勤1名
- ビジネス客多め・部屋数が多い日：8時出勤2名または8時1名、10時1名の2名体制

■ 繁忙期（4〜5月・10〜11月）
- 基本：8時出勤1名＋10時出勤1名（2名体制）
- 満室かつチェックアウトが集中する日：
　8時・10時・12時に各1名（3名体制）
　※チェックアウトが9〜10時に集中してチェックインに間に合わないリスクを回避するため

### 週次シフト設計の運用
1週間毎に翌週の予約状況を確認しシフトを設計する。
【理由】予約状況は1週間単位で比較的安定しており、シフト設計の精度を保ちやすいため。

※予約状況・客層比率の変動に応じてシフトを柔軟に調整することを前提とする。

週次シフト設計には、PMSから翌週分の予約データをエクスポートし
Looker Studio等のBIツールで可視化する運用を提案。
直近1週間の客層・チェックアウト時間の傾向を
自動レポート化することでより効率的なシフト設計が可能になる。

### イレギュラー予約対応
急なキャンセル・客層変動が生じた場合は臨機応変に対応する。

・キャンセルで部屋数が減った場合
　→ 余剰スタッフはイレギュラー業務
　　（普段手が回らない箇所の清掃等）に充てる。

・急な予約増加の場合
 　→ 清掃が完了しないリスクがあるため、予備要員の確保または
 チェックインまでの清掃時間調整を行う。
